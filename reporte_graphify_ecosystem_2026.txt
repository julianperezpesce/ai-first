Resumen Ejecutivo
safishamsi/graphify (42.6K estrellas, MIT, Python) transforma cualquier carpeta en un grafo de conocimiento con-
sultable en un comando, eliminando el desperdicio de tokens de los asistentes AI que usan grep. Creado el 3 de abril
de 2026, ha alcanzado 42.6K estrellas en solo 31 dias.
Este reporte analiza los 15 mejores proyectos open-source de 2026 que complementan a Graphify en tres dimen-
siones criticas para workflows AI:
•    RAG Local: ChromaDB, FAISS, LlamaIndex — recuperacion semantica privada
•    Indexacion de Codebases: busqueda semantica de codigo con tree-sitter + vectores
•    Skills Claude/Anthropic: Ecosistema MCP, plugins, y agentes autonomos
Cada proyecto incluye estrellas reales, commits recientes y guia de integracion practica.                                                         -2-

           Estrellas:          42,622
           Lenguaje:           Python 100%

Pipeline
detect() -> extract() -> build graph() -> cluster() -> analyze() -> report() -> export()
Tres pasadas de extraccion:
•    AST (local, costo API cero): tree-sitter parsea 26 lenguajes
•    Transcripcion (local): faster-whisper para video/audio
•    Semantica (LLM): docs, PDFs, imagenes a Claude/Kimi para extraccion de conceptos
16 plataformas integradas: Claude Code, Codex, OpenCode, Cursor, Gemini CLI, Copilot CLI, VS Code, Aider y
mas. Se instala como skill con un comando.
Que le falta a Graphify (brechas que cubren los proyectos complementarios):
•    Sin almacenamiento vectorial (usa Leiden topologico, no embeddings)
•    Sin backend RAG persistente multi-proyecto
•    Sin indexacion semantica a nivel de simbolo (solo AST)
•    Sin integraciones SaaS/MCP externas
•    Sin sandbox de ejecucion de agentes                                                        -3-

6. Guia de Integracion

6.1 Stack Recomendado para Workflow AI Local
        Graphify + LightRAG + Chroma + Vera
•   Graphify: Mapa estructural del codebase (grafo de conocimiento)
•   LightRAG: RAG sobre documentacion y specs (grafo desde docs)
•   Chroma: Backend vectorial compartido para ambos
•   Vera: Busqueda precisa de codigo (BM25+vector+reranker)
             Graphify + LightRAG + Chroma + Continue + Composio + anthropics/skills
•   Graphify + LightRAG + Chroma: Nucleo de conocimiento
•   Continue: Verificacion automatizada en CI/CD con agentes MCP
•   Composio: Conexion a SaaS (GitHub, Slack, Notion, etc.)
•   anthropics/skills: Skills oficiales para Claude Code
             Graphify + LightRAG + Chroma + txtai + Continue + Composio + anything-llm
             + antigravity-awesome-skills
Todos los stacks son 100% locales excepto las integraciones SaaS via Composio y las llamadas LLM API.

6.2 Integraciones Especificas
Graphify no usa vectores internamente. Sus nodos (funciones, clases, archivos) pueden enriquecerse con embed-
dings en Chroma:
        # 1. Graphify genera el grafo
        graphify .

        # 2. Extraer nodos clave del graph.json
        # Para cada nodo, generar embedding y almacenar en Chroma
        # Usar chromadb.Client() con embedding function de Ollama
LightRAG construye KG desde documentos; Graphify desde codigo. Union via Neo4j:
        # Graphify exporta a Neo4j
        graphify . --neo4j

        # LightRAG inserta documentos al mismo Neo4j
        lightrag insert docs/ --neo4j

        # Ambos grafos coexisten en la misma DB
        # Consultas Cypher cruzan codigo y documentacion
Graphify da contexto estructural, Vera encuentra el codigo exacto:                                       -10-


        # 1. Graphify genera el mapa del proyecto
        graphify .

        # 2. El agente lee GRAPH_REPORT.md para entender estructura

        # 3. Luego usa Vera MCP para busqueda precisa
        # Vera tool: search_code("auth middleware pattern")
Instalacion conjunta en Claude Code:
        # 1. Instalar Graphify
        graphify install

        # 2. Instalar skills oficiales
        claude plugins add anthropics/skills

        # 3. Instalar coleccion comunitaria
        npx antigravity-awesome-skills

        # Claude Code ahora tiene:
        # - Graphify: mapa estructural del codebase
        # - anthropics/skills: skills oficiales de Anthropic
        # - antigravity: 1,445+ skills comunitarias
        # .continue/checks/graphify-check.md
        # Continue ejecuta este agente en cada PR:

        # 1. Ejecuta "graphify extract" en el branch del PR
        # 2. Compara el grafo con el de main
        # 3. Detecta cambios estructurales sospechosos
        # 4. Reporta en el PR como status check                                                         -11-
